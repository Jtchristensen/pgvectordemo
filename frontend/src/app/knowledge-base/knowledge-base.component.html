How it works

1
Text Extraction

Your file is read and text is extracted (supports PDF & plaintext)

2
Chunking

Text is split into ~600-character overlapping chunks for better retrieval

3
Embedding

Each chunk is converted to a 768-dimensional vector using nomic-embed-text

4
pgvector Storage

Vectors are stored in PostgreSQL with HNSW index for fast cosine similarity search
